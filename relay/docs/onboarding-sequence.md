## PART 2: EMAIL DESIGN RECOMMENDATIONS

### Plain Text vs HTML for Freemium Onboarding[42][43][44][46]

**Research Findings:**

1. **Plain text emails outperform HTML:**
   - Plain text: 21-42% higher click-through rate[44]
   - Plain text: 25-42% higher open rate (when comparing to heavy HTML)[49]
   - Plain text: Drop in opens of 23-37% when images added to HTML[42][43]

2. **Why plain text wins for onboarding:**[46]
   - Looks more personal ("handwritten feel")[44]
   - Better deliverability (avoids spam filters)
   - Less distraction from main message
   - Mobile-friendly (renders perfectly on all devices)
   - Founder emails feel more authentic[47]

3. **When to use HTML (secondary use):**
   - Branded promotional emails (day-14 upgrade email)
   - When aesthetic consistency matters more than conversion
   - If audience already expects HTML design

**Recommendation for Relay:** Use plain text for emails 1-3 (onboarding/education), light HTML for email 5 (upgrade, branded).

---

## PART 3: BEHAVIORAL TRIGGERS & EVENT-BASED EMAILS

### Behavioral Trigger Strategy[52][53][54][57][59]

Behavioral trigger emails outperform time-based drip campaigns. Instead of sending all emails on a fixed schedule, send emails based on user actions (or lack of action).

**Key Principle:** Trigger-based emails yield significantly better engagement rates[35]

---

## OPTIMIZATION ROADMAP

**Month 1:** Launch baseline emails, measure metrics, identify drop-off points

**Month 2:** A/B test:
  - Subject line variations (data-driven vs. benefit-driven)
  - Send times (morning 8am vs. afternoon 2pm)
  - CTA copy ("Create Invoice" vs. "Get Started")

**Month 3:** Optimize based on data:
  - Improve Email 2 tutorial if invoice creation rate < 20%
  - Strengthen Email 3 social proof if click rate < 4%
  - Adjust Email 5 upgrade prompt timing if conversion < 5%

**Ongoing:** Monthly analysis of:
  - Conversion funnel (emails ‚Üí invoice created ‚Üí Pro upgrade)
  - Unsubscribe rates (if > 0.5%, reduce email frequency)
  - Spam complaints (if > 0.1%, revisit sender name/subject line)
